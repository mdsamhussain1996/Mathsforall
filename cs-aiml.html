Vector Spaces, Bases, and Span

A complete module on the fundamental architecture of linear algebra

🔢
1. Vectors in \(\mathbb{R}^n\)
The building blocks of space
▾

A vector in \(\mathbb{R}^n\) is an ordered n-tuple: \(\mathbf{x} = (x_1, x_2, \dots, x_n)\). This can be interpreted as a **point** in space or an **arrow** from the origin.

Common Spaces:
  • \(\mathbb{R}^1\): Real line
  • \(\mathbb{R}^2\): 2D Plane (pairs)
  • \(\mathbb{R}^3\): 3D Space (triples)
🏗️
2. Vector Spaces & Axioms
Formal mathematical definition
▾

A set \(V\) with addition and scalar multiplication is a vector space if it satisfies 10 axioms:

1. Closure (Add)
\(\mathbf{u}+\mathbf{v} \in V\)
2. Commutativity
\(\mathbf{u}+\mathbf{v} = \mathbf{v}+\mathbf{u}\)
3. Associativity (Add)
\((\mathbf{u}+\mathbf{v})+\mathbf{w} = \mathbf{u}+(\mathbf{v}+\mathbf{w})\)
4. Zero Vector
\(\mathbf{u}+\mathbf{0} = \mathbf{u}\)
5. Additive Inverse
\(\mathbf{u}+(-\mathbf{u}) = \mathbf{0}\)
6. Closure (Scalar)
\(c\mathbf{u} \in V\)
7. Distributivity (Vec)
\(c(\mathbf{u}+\mathbf{v}) = c\mathbf{u} + c\mathbf{v}\)
8. Distributivity (Scl)
\((c+d)\mathbf{u} = c\mathbf{u} + d\mathbf{u}\)
9. Associativity (Scl)
\(c(d\mathbf{u}) = (cd)\mathbf{u}\)
10. Scalar Identity
\(1\mathbf{u} = \mathbf{u}\)
🧱
3. Subspaces
Spaces within spaces
▾

A subset \(W \subseteq V\) is a subspace if it is closed under addition and scalar multiplication. Geometrically, this means it must pass through the origin.

\text{If } \mathbf{u, v} \in W, \text{ then } (\mathbf{u}+\mathbf{v}) \in W \text{ and } c\mathbf{u} \in W
🔗
4-5. Linear Combination & Span
The reach of vectors
▾

A **linear combination** is a sum of vectors scaled by constants: \(\mathbf{v} = c_1\mathbf{v}_1 + c_2\mathbf{v}_2 + \dots + c_k\mathbf{v}_k\).

The **Span** is the set of all possible linear combinations. It defines the entire geometric space reachable by the set of vectors.

\[\text{span}(S) = \left\{ \sum_{i=1}^{k} c_i \mathbf{v}_i \;\middle|\; c_i \in \mathbb{R} \right\}\]

In simpler terms: if you have two non-parallel vectors in 3D space, their span is the 2D plane they "draw" together.

💎
6-8. Independence, Basis & Dimension
The core of representation
▾

Linear Independence: A set is independent if no vector can be written as a combination of others. Formally: \(\sum c_i \mathbf{v}_i = \mathbf{0} \implies c_i = 0\) for all \(i\).

Basis: A set that is both linearly independent and spans the space.

Dimension: The number of vectors in any basis for the space.

📜
9. Important Theorems
Key theoretical results
▾
  • Uniqueness: Each vector has a unique representation in a given basis.
  • Basis Size: All bases for a given vector space have the same number of elements (dimension).
  • Dependence: Any set with more vectors than the dimension of the space is linearly dependent.
🌀
10. Eigenvalues & Eigenvectors
Principal directions of transformation
▾

An **eigenvector** is a special vector that only changes its scale (not direction) when a linear transformation is applied.

\[A\mathbf{v} = \lambda\mathbf{v}\]

Where \(\lambda\) is the **eigenvalue**. This is critical for **PCA** and stability analysis in control theory.

Application of Eigenvalue and Eigenvector

Eigenvalues and eigenvectors are fundamental in AI/ML for reducing data complexity and identifying key structures. They are primary tools for:

  • Dimensionality Reduction: Principal Component Analysis (PCA) uses eigenvectors to find the directions of maximum variance.
  • Image Compression: Singular Value Decomposition (SVD) uses them to compress matrices while retaining core features.
  • Face Recognition: Eigenfaces leverage them to extract the most important facial features.
  • Recommendation Systems: Latent factor analysis uses them to discover hidden patterns.
  • Graph Analysis: Spectral clustering and PageRank use them to rank nodes and find community structures.

The Expert View: Eigenvectors in AI/ML

📐PART 1 — MATHEMATICAL FOUNDATION

Eigenvalues (\(\lambda\)) and Eigenvectors (\(\mathbf{v}\)) are defined by the fundamental equation: \[\mathbf{A}\mathbf{v} = \lambda\mathbf{v}\]

  • Geometric Meaning: A matrix \(\mathbf{A}\) typically rotates and stretches vectors. An eigenvector is a special direction where the matrix only stretches or compresses it. The scale factor is the eigenvalue.
  • Derivation: Rewriting as \((\mathbf{A} - \lambda\mathbf{I})\mathbf{v} = 0\), for a non-zero \(\mathbf{v}\), the matrix must squish space, meaning its determinant is zero: \[\det(\mathbf{A} - \lambda\mathbf{I}) = 0\]
  • Properties: The trace (sum of diagonals) equals \(\sum\lambda_i\), and the determinant equals \(\prod\lambda_i\).
  • Covariance Matrices: In ML, covariance matrices \(\mathbf{C} = \frac{1}{n}\mathbf{X}^T\mathbf{X}\) are always symmetric and positive semi-definite. Symmetric matrices always have real eigenvalues and orthogonal eigenvectors.

📉PART 2 — Principal Component Analysis (PCA)

PCA finds axes of maximum variance by eigendecomposing the covariance matrix.

PCA Scree Plot (Variance Explained)

Magnitude of eigenvalues \(\lambda_i\) corresponds to the variance captured by each principal component.

  • Pipeline: Center data → Compute \(\mathbf{C} = \frac{1}{n}\mathbf{X}^T\mathbf{X}\) → Eigendecompose → Project.
  • Example: MNIST (784 dims → ~154 dims for 95% variance).

🖼️PART 3 — Singular Value Decomposition (SVD)

SVD factorizes any matrix: \[\mathbf{A} = \mathbf{U}\mathbf{\Sigma}\mathbf{V}^T\]

Singular Value Decay

The "Elbow" in the singular value curve tells us the optimal rank \(k\) for compression.

  • Eckart-Young Theorem: Rank-\(k\) SVD is the optimal low-rank approximation.
  • Storage Math: Compressed size = \(k(m + n + 1)\).

👤PART 4 — Eigenfaces & Face Recognition

Recognizing faces by projecting them into "Face Space" spanned by eigenvectors.

  • Computational Trick: Solving the \(N \times N\) inner product matrix instead of the massive pixel covariance matrix.
  • Face Space: Projection \(\mathbf{w} = \mathbf{U}_k^T(\mathbf{x} - \mu)\).

🎬PART 5 — Recommender Systems (Latent Factors)

Discovering hidden patterns in user tastes through matrix factorization.

  • Latent Factors: SVD reveals concepts like "genre" or "actor preference" as eigenvectors.
  • Netflix Prize: Achieved 10% improvement using \(k=40\) latent factors.

🕸️PART 6 — Graph Analysis: PageRank

PageRank is the dominant eigenvector of the web graph transition matrix: \[\mathbf{P}\mathbf{\pi} = \mathbf{\pi}\]

  • Spectral Clustering: Uses the 2nd smallest eigenvector of the Graph Laplacian (Fiedler vector) to find optimal cuts.

💡 AI Application

In **Machine Learning**, "Embeddings" (like Word2Vec) are vectors in a high-dimensional vector space. **Basis transformation** (like PCA) helps us find the most important "latent" directions in our data, reducing noise and complexity.

Example 1: Independence

Check if \(\mathbf{v}_1=(1,2)\) and \(\mathbf{v}_2=(2,4)\) are independent.

Since \(\mathbf{v}_2 = 2\mathbf{v}_1\), one is a multiple of the other. They are linearly dependent.

Example 2: Basis Find

Find a basis for \(W = \{ (a, b, a+b) \mid a,b \in \mathbb{R} \}\).

\((a, b, a+b) = a(1,0,1) + b(0,1,1)\). The vectors \((1,0,1)\) and \((0,1,1)\) span \(W\) and are independent. Basis found!

Example 3: Polynomial Space

Check if \(1, x, x^2\) form a basis for \(P_2\).

Yes. They are linearly independent (no power of \(x\) can be formed by others) and they span all quadratic polynomials. This is the **standard basis** for \(P_2\).

Systems of Linear Equations

Foundational techniques for solving simultaneous constraints

The Core Concept

A system of linear equations is a set of two or more equations with the same variables. The solution is the ordered pair \((x, y)\) that satisfies all equations. In AI, systems of equations are the basis for understanding how multiple features interact and how weights are optimized.

Methods for Solving

  • 5.1 Graphing: Visualizing the intersection of two lines.
  • 5.2 Substitution: Isolating one variable and plugging it into the other.
  • 5.3 Elimination: Adding or subtracting equations to remove a variable.
  • 5.4 Special Cases: Identifying No Solution (parallel) vs. Infinitely Many (same line).

Mastering these foundational steps is essential before moving to higher-dimensional Vector Spaces and Matrix Operations.

Probability & Statistics

Bayes theorem, distributions, and statistical inference

Bayes' Theorem

The most important theorem in probabilistic reasoning. It tells us how to update our beliefs when we receive new evidence. Every Bayesian neural network, spam filter, and medical diagnosis tool is built on this foundation.

P(A|B) = P(B|A) · P(A) / P(B) posterior = likelihood × prior / evidence

Probability Distributions

Distributions describe how probability is spread over values. The Gaussian (Normal) distribution appears everywhere due to the Central Limit Theorem. In ML, we model latent variables, noise, and predictions as distributions.

N(x; μ, σ²) = (1/√(2πσ²)) · exp(-(x-μ)²/2σ²)

Maximum Likelihood Estimation

We find model parameters that maximize the probability of observing our training data. This directly connects probability theory to how we train neural networks (cross-entropy loss = negative log-likelihood).

  • Random variables & expectation
  • Bayes' theorem & conditional probability
  • Normal, Bernoulli, Poisson distributions
  • MLE & MAP estimation
  • Central Limit Theorem
  • Information theory & entropy

Optimization

Gradient descent, convex optimization, and learning algorithms

Gradient Descent

The algorithm that trains virtually every neural network. We compute the gradient of the loss function with respect to our parameters, then take a small step in the opposite direction — downhill on the loss landscape.

θ ← θ − α · ∇_θ L(θ) Stochastic: θ ← θ − α · ∇_θ L(θ; xᵢ, yᵢ)

Convex Optimization

When the loss landscape has a single bowl shape (convex), gradient descent guarantees finding the global minimum. Support Vector Machines (SVMs) solve a convex optimization problem. Understanding convexity helps explain why some ML problems are easier than others.

Adam Optimizer

Adaptive Moment Estimation combines momentum (remembering past gradients) with per-parameter learning rates. It's the most widely used optimizer in deep learning — automatically adapting the learning rate for each weight.

mₜ = β₁mₜ₋₁ + (1-β₁)∇θ vₜ = β₂vₜ₋₁ + (1-β₂)(∇θ)² θ ← θ − α · m̂ₜ/√(v̂ₜ+ε)
